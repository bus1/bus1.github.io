Description

The bus1-client object is a small, direct wrapper of the bus1 kernel API. It allows direct access to all bus1 client features, but additionally provides a limited set of helpers to avoid dealing with API extensions, client pools, and ioctl calling conventions.

The bus1-client API is designed for threaded access. Apart from constructors and destructors, any function can be called in parallel from multiple threads without any synchronization necessary (neither internally nor externally). They map 1-to-1 to the kernel API, but hide the ioctl marshaling. Furthermore, the API is designed to allow *multiple* different contexts on the same file-descriptor, without knowing about each other.
